# Shippable-readiness ledger

This ledger is the single honest accounting of what stands between
WOHJO and ship. Every gate item is one of:

- **Jobs** — proven, with the artefact named. The thing has been
  demonstrated, not asserted.
- **should-work** — likely true, never proven; cannot ship on this.
- **not-started** — the work has not begun.
- **needs-Lauren** — blocked on an action only the human owner can take
  (production write, merge authorisation, secret provisioning,
  org-only access).
- **needs-chat-Claude** — blocked on a per-dimension attestation
  against live production that the autonomous CI session cannot
  perform itself.

"Shippable" is the state where every P0 row is Jobs, needs-Lauren, or
needs-chat-Claude with the latter two scheduled. It is not declared by
this file — this file just makes it knowable.

Last revised: 2026-06-09. Source of truth: the PR / commit / artefact
named in the Proof column.
