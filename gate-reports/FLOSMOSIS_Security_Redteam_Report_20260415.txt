FLOSMOSIS SECURITY RED TEAM REPORT
Date: 2026-04-15
Scope: 8 attack surfaces

================================================================
SURFACE 1: AUTH — Missing authentication on API routes
================================================================
SEVERITY: CRITICAL (deferred to Sprint 5)

FINDINGS:
- /api/field/worker — unauthenticated GET, returns worker data by phone
- /api/field/shift/start — unauthenticated POST, any worker_id accepted
- /api/command/* — no session validation on command endpoints
- /api/verify/* — supervisor_id taken from query param, not session

STATUS: Known issue. Full auth hardening scheduled for Sprint 5
(security sprint before Apr 28 pilot). Currently mitigated by:
- App not publicly advertised yet
- Supabase RLS blocks direct DB access
- Service role key server-side only (verified)

================================================================
SURFACE 2: IDOR — Insufficient access control
================================================================
SEVERITY: HIGH (deferred to Sprint 5)

FINDINGS:
- /api/command/shifts/[shiftId]/approve — accepts admin_user_id from body
- /api/verify/shifts — supervisor_id from query param, not verified
- No company_id scoping on most queries

STATUS: Sprint 5 will add company_id validation derived from session.

================================================================
SURFACE 3: WLES Integrity — Hash chain security
================================================================
SEVERITY: PASS

FINDINGS:
- SHA-256 hash chain correctly implemented
- generateEventHash uses deterministic input ordering
- verifyHashChain validates both hash correctness and chain linkage
- GENESIS sentinel accepted for first event
- shift_events table is immutable (INSERT only)

STATUS: No issues found.

================================================================
SURFACE 4: Twilio — Webhook signature validation
================================================================
SEVERITY: PASS

FINDINGS:
- /api/webhooks/twilio/sms-reply validates Twilio signature first
- TWILIO_AUTH_TOKEN server-side only
- No Twilio credentials in client-side code

STATUS: No issues found.

SURFACE 5: Financial Data — Pay rate exposure
================================================================
SEVERITY: HIGH — FIXED

FINDINGS:

================================================================

SEVERITY: HIGH (deferred to Sprint 5)

SURFACE 8: Infrastructure — Credential exposure
================================================================
SEVERITY: PASS

FINDINGS:

- SUPABASE_SERVICE_ROLE_KEY only imported in server.ts
